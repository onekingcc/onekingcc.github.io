Apache Commons-Collections 简介

Apache Commons Collections是一个扩展了Java标准库里的Collection结构的第三方基础库，它提供了很多强有力的数据结构类型并且实现了各种集合工具类。作为Apache开源项目的重要组件，Commons Collections被广泛应用于各种Java应用的开发。

一些Java应用程序(Weblogic，Websphere，Jboss，Jenkins，Coldfusion等)的RCE漏洞都是因为Commons-Collections的反序列化造成的。

第一次对java反序列化漏洞的第一篇利用链的文章，所以想把这其中的学习过程记录下来，前面的很多前辈已经将这个系列分析的很多了，本人只是基于学习的角度，进行调试跟踪漏洞。

反射调用Runtime实现本地命令执行的流程如下：

  1. 反射获取Runtime类对象（Class.forName(“java.lang.Runtime”)）

  2. 使用Runtime类的Class对象获取无参数的构造方法getDeclaredConstructor()，因为Runtime()的构造方法为私有，所以需要利用反射去修改里面的属性值constructor.setAccessible(true)

  3. 获取Runtime类的exec(String)方法（runtimeClass1.getMethod(“exec”, String.class);)

  4. 最后进行调用exec(String)方法。

漏洞分析

Apache Commons Collections，中提供了一个transform类，这个类功能就是将一个对象转换为另外一个对象。

漏洞定位：

org/apache/commons/collections/functors/InvokerTransformer.java中的transform方法：

这段代码很容易看到这里使用了反射，执行input对象的一个方法，查看一下该方法来源发现是实例化InvokerTransformer传入的methodName

这里可以得出这段代码调用的方法名和Class对象是可控的，这样我们直接就就可以在这个包中全局进行搜索该方法，看哪里用到该方法。

在org/apache/commons/collections/functors/ChainedTransformer.java

这里发现是反复遍历Transformer元素的transform方法，然后返回object，返回的object的结果又当成下一次调用的参数执行。利用这一点可以形成一个反射的利用链。

Transformer的元素为InvokeTransforms类的对象，那就可以调用InvokeTransform类里面的方法，往上进行回溯，在org/apache/commons/collections/map找到LazyMap类的transform方法，这里会先去判断当前的map中是否存在key，如果没有就会由factory。transform()进行处理，然后跟进factory

发现这里的factory是不能控制的，这里的factory需要通过LazyMap中的decorate方法对其进行一个赋值。

可以看到这里会初始化factory，同时会实例化LazyMap。那么现在需要去找一个类，在对象进行反序列化时会调用我们构造对象的get(Object)方法

org/apache/commons/collections/keyvalue/TiedMapEntry.java

这里可以发现在TiedMapEntry.java中存在一个getValue()方法，进入方法发现会去调用get()方法，但是getValue()方法在当前类中toString 中被调用、

在/Library/Java/JavaVirtualMachines/jdk1.8.0_202.jdk/Contents/Home/src.zip!/javax/management/BadAttributeValueExpException.java中发现有2处toString()方法,toString()方法是在对象被当作字符串操作时，自动被调用，有一处是构造方法中，还有一处是在readObject中，在readObject（）进行了重写，然后调用了toString() 方法,大多数java反射机制和反序列化利用点readObject()

那么这里差不多可以得到pop链

BadAttributeValueExpException.readObject->TiedMapEntry.toString->TiedMapEntry.getValue->LazyMap.get->ChainedTransformer.transform-> InvokerTransformer.transform

这里需要注意的就是，pop链中很多都是基于面向属性编程，默认对象是可控的，但是很多这里的val值是不可控的所以需要如下方法进行设置setaccessible为true。
